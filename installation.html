v1.0.1 Installation instructions

MacOS Installation Instructions

Open the dmg file and drag the CARTA-v1.0.1.app to the /Applications folder.
To run CARTA-v1.0.1, click its icon in the Launchpad, or double click it in the Finder.
Alternatively, the user may run CARTA from the terminal by adding an alias to their ~/.bashrc file. e.g. alias carta=/Applications/CARTA-v1.0.1.app/Contents/MacOS/CARTA-v1.0.1 and source ~/.bashrc.
Now CARTA can be started by typing carta in the terminal.

Linux Installation Instructions (local version)

The RedHat7 and Ubuntu versions are provided as AppImages.
After downloading, make the AppImage file executable
chmod 755 CARTA-v1.0.1-ubuntu.AppImage and run it
./CARTA-v1.0.1-ubuntu.AppImage

On RedHat7 machines, an updated 'nss' package may need to be installed; sudo yum install nss
Also on RedHat7 machines after starting the AppImage, you may see a warning about 'Fontconfig'. It does not affect using CARTA, but the warning can be removed with sudo yum install fontconfig

Linux Installation Instructions (remote version)

On your remote server, extract the archive tar -xvf CARTA-v1.0.1-RedHat6-RedHat7-remote.tar.gz and run the carta script within ./CARTA-v1.0.1-RedHat6-RedHat7-remote/carta.
This will provide a unique URL. Copy and paste this URL in to your local web browser to access CARTA. When finished, close your browser window, and press 'q' in the terminal window to close CARTA.
The default search path for images is $HOME, the remote version has way to override this by appending a path to a directory of your choice e.g../CARTA-v1.0.1-RedHat6-RedHat7-remote/carta /path/to/my/images .

For more detailed installation instructions, please consult the Installation and Configuration section of the CARTA user manual here.
